# Security Policy

## Reporting a vulnerability

Please do not open public issues containing secrets, exploit details, private source code, or proprietary project files.

If you find a security issue in this template, report it privately using GitHub's private vulnerability reporting if available, or contact the repository owner.

## Sensitive data rules

Do not upload:

- API keys
- bot tokens
- OAuth secrets
- database connection strings
- private source code
- purchased source code
- generated mirror docs from private projects
- logs containing secrets

## Supported versions

Only the latest version of this template is actively maintained.

## Scope

This project is a workflow/template. Security reports should relate to:

- accidental secret exposure risks
- unsafe `.gitignore` behavior
- prompts that may encourage copying private source
- helper scripts that may leak private project data
- unsafe generated-output handling
