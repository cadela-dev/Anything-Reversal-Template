# Status Legend

- `[ ]` = not assigned
- `[~]` = assigned / in progress
- `[x]` = mirrored documentation complete and verified
- `[D]` = deferred/skipped with explicit reason
- `[!]` = error / human review needed

Only verifier marks `[x]`.

Every `[D]` entry must include a clear reason.

Every `[!]` entry must include a clear repair/review reason.

No file disappears from the index without explanation.
